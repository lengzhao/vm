# 智能合约安全审查规范

## 1. 概述

本文档定义了智能合约虚拟机项目中合约代码的安全审查规范，确保所有执行的合约代码符合安全要求。为了复用Golang编译器和TinyGo编译器，并考虑到不同版本的兼容性问题，关键字和默认函数采用白名单机制。

## 2. 审查流程

### 2.1 AST解析
在合约编译阶段，系统会对合约源代码进行AST（抽象语法树）解析，提取代码结构信息用于后续审查。

### 2.2 关键字检查
系统会检查代码中是否包含黑名单中的关键字，如果发现则标记合约发布失败。

### 2.3 导入列表检查
系统会检查合约的导入列表，确保只导入允许的安全库。

## 3. 关键字白名单

为了确保不同Golang版本的兼容性并复用现有编译器，采用白名单机制定义允许的关键字，包括但不限于：

- 基本类型关键字：`int`, `int8`, `int16`, `int32`, `int64`, `uint`, `uint8`, `uint16`, `uint32`, `uint64`, `float32`, `float64`, `complex64`, `complex128`, `string`, `bool`, `byte`, `rune`, `uintptr`
- 常量定义：`const`, `iota`
- 变量定义：`var`
- 类型定义：`type`, `struct`, `interface`, `func`
- 控制流：`if`, `else`, `for`, `switch`, `case`, `default`, `break`, `continue`, `fallthrough`, `return`
- 包管理：`package`, `import`
- 其他安全关键字：`nil`, `true`, `false`

## 4. 允许导入的包白名单

合约只能导入以下指定的安全库：

- `fmt`: 格式化输入输出
- `strconv`: 字符串转换
- `math`: 数学计算
- `time`: 时间处理
- `errors`: 错误处理
- `github.com/lengzhao/vm`: 项目默认库，提供与区块链环境交互的接口

## 5. 默认函数白名单

为了确保合约的安全性和兼容性，只允许使用以下默认函数：

- 基本类型转换函数：`int()`, `int8()`, `int16()`, `int32()`, `int64()`, `uint()`, `uint8()`, `uint16()`, `uint32()`, `uint64()`, `float32()`, `float64()`, `complex64()`, `complex128()`, `string()`, `bool()`, `byte()`, `rune()`
- 字符串处理函数：`len()`, `append()`, `copy()`, `delete()`
- 内存操作函数：`make()`, `new()`
- 错误处理函数：`error()`, `Error()`
- 项目默认库提供的函数

## 6. 审查失败处理

当合约代码未能通过安全审查时，系统将：
1. 标记合约发布失败
2. 返回详细的错误信息，包括违反的安全规则
3. 阻止合约进入编译和执行阶段
